# IMPL

This folder is where agents place concrete implementation planning artifacts.

## File conventions
- Epic files: `EPIC-<nn>-<short-name>.md`
- Story files: `STORY-<epic>-<nn>-<short-name>.md`
- Sprint files: `SPRINT-<nn>.md` (optional, if split from `SPRINT_PLAN.md`)

## Update rules
- do not rewrite completed story history; append status updates
- every story must include slice checklist status
- every sprint close must record what passed/failed at integration gate

## Starting point
- `EPIC_BACKLOG.md`
- `SPRINT_PLAN.md`
- `STORY_SLICE_TEMPLATE.md`
- `STORY_SPEC_QUEUE.md`
- `HANDOFF_LOG.md`
- `AGENT_COMMUNICATION_PROTOCOL.md`
